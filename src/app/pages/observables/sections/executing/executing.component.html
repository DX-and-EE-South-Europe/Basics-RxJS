The code inside the Observable represent the Obervable execution.

As you see in 'creating observable' section, an observable emits three different type of values:

  • Next: When calling the 'next' method, it emits some value like a string, number, array, object...
  • Error: When an Error happens inside an observable, it emmits the error or exception.
  • Complete: When the 'complete' method is called inside the observable. (It does not emit any value).

'Next' is the most common and important type of value emitted by an observable. 'Error' and 'Complete' only happen once and they are like a trigger that stops the execution of the observable.

To manage the values emitted by the observable, we can pass arguments to the subscribe method:

  • 1 Callback: It executes only the values emitted by the 'next' method.
    • {{ executing.exampleCallback }}
  • 2 Callback:The first callback it executed only the values emitted by the 'next' method, and the second when an error is caught. 
    • {{ executing.example2Callback }}
  • 3 Callback: The first is for the 'next' method, the second for the 'error', and the third for the 'complete'.
    • {{ executing.example3Callback }}
  • Observer (Best choice): A collection of callbacks that listen the differents types of values emmited by the observables. 
    • {{ executing.exampleObserver }}

This app use a custom observer for the demos examples.
