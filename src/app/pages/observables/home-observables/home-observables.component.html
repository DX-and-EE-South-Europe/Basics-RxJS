Observables:

  • Are an information source.
  • Can emit zero, one or more values.
  • Are synchronous or asynchronous.
  • Are finites or infinites.
  • Use a push system.

To sum up, they are not Promises or EventEmitters. They are like a lazy functions that need to be called, do not share side effects, emit zero, multiple or infinite values synchronously or asynchronously.

They really usefull to manage UI events, notify changes objects, sockets comunication, work with streams...

Some of their benefits are:

  • Avoid callback hell (a callback inside another callback).
  • Easier and simplified work.
  • Easier to transform streams (operators).
  • Cleaner and easier to read.
  • Easier to implement and add procedures without changing the final product.

In this section, you will learn about the different types of values that an observable can emit, how to execute and interact with them.

Note: It is a convention to indicate when something is an Observable the '$' symbol should be added at the end of the name. Also, you can put between <> the type of value that the observable emits.
